# 一、python中的"="原理
在 Python 中，= 不是“赋值”操作（像 C 或 Java 那样把值复制过去），而是“绑定”（binding）或“引用”（reference）操作。这是理解 Python 对象模型的关键！
🧠 核心原理：变量是对象的标签（名字），不是容器
在 Python 中：
- 所有东西都是对象（int, str, list, 自定义类实例等）。
- 变量（名字）只是指向这些对象的“标签”或“引用”。
- = 的作用是：把一个名字（变量）绑定到某个对象上。

举个简单例子：
```python
x = 100
y = x
```
执行过程：
- 创建一个整数对象 100（如果内存中还没有的话）。
- 把名字 x 绑定到这个 100 对象。
- 把名字 y 也绑定到同一个 100 对象。
```python
print(id(x))  # 比如 140712345678904
print(id(y))  # 一样！140712345678904
print(x is y) # True
```
✅ 这里 x 和 y 是两个名字，指向同一个对象。
1. 不可变对象

整数、字符串、元组等都是不可变对象。一旦创建，值就不能改变。

对可变对象（如 list）的影响：
```python
a = [1, 2, 3]
b = a         # b 和 a 指向同一个列表对象
b.append(4)
print(a)      # [1, 2, 3, 4] ← 被改变了！
```
改变内容不会改变位置
因为 a 和 b 都是指向同一个列表对象，修改对象内容，所有引用都会看到变化。
再赋值会发生什么？
```python
a = [1, 2, 3]
b = a
a = [4, 5, 6]  # 这里不是修改原列表，而是让 a 指向一个新列表！
print(b)       # [1, 2, 3] ← 没变
```
a = [4,5,6] 创建了一个新列表对象，然后把名字 a 重新绑定到这个新对象。
b 仍然指向原来的 [1,2,3]。
📌 关键：= 永远是“绑定名字到对象”，不会修改已有对象（除非你用 +=, .append() 这类方法）。
💡 类比理解（推荐！）
想象对象是“气球”，变量是“绳子”：

x = [1,2,3] → 吹一个气球（列表对象），用绳子 x 系住它。
y = x → 再拿一根绳子 y，系到同一个气球上。
x.append(4) → 气球变大了（内容变了），两根绳子都连着它。
x = [4,5] → 吹一个新气球，把 x 这根绳子改系到新气球上，y 还在原来的气球上。
